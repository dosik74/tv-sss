### 3. React Компонент - `src/components/VideoPlayer.tsx`

**Новый полнофункциональный компонент с:**

#### Интерфейс
```typescript
interface VideoPlayerProps {
  id: string;                    // ID контента
  idType: string;                // Тип ID (kinopoisk, shikimori и т.д.)
  title: string;                 // Название аниме
  initialEpisode?: number;       // Стартовая серия
  episodesTotal?: number;        // Всего серий
  onClose?: () => void;          // Callback при закрытии
}
```

#### Основные функции
1. **HLS.js видеопотоком**
   - Автоматическое выявление адаптивного битрейта
   - Обработка ошибок потока

2. **Управление озвучками**
   - Загрузка доступных озвучек при открытии
   - Меню выбора озвучки в контролах
   - Перезагрузка видео при смене озвучки

3. **Управление сериями**
   - Полный список серий с номерами и названиями
   - Индикатор текущей серии
   - Прогресс просмотра для каждой серии
   - Сохранение позиции при переключении серий

4. **Контролы плеера**
   - Play/Pause с большой кнопкой в центре
   - Skip Previous/Next (также по клавишам P/N)
   - Прогресс-бар с отображением времени
   - Громкость со слайдером
   - Скорость воспроизведения (0.5x - 2x)
   - Полноэкранный режим

5. **Клавиатурное управление**
   - `Space` - Play/Pause
   - `Arrow Right/Left` - ±10 секунд
   - `Arrow Up/Down` - Громкость
   - `M` - Mute/Unmute
   - `F` - Fullscreen
   - `N` - Следующая серия
   - `P` - Предыдущая серия
   - `Esc` - Закрыть меню/панели

## 🔌 API Endpoints

### GET /episodes
```
Параметры:
  - id: string (ID контента)
  - id_type: string (kinopoisk, shikimori, imdb, kodik)

Ответ:
{
  "episodes": [ { "number": 1, "title": "Серия 1" }, ... ],
  "series_count": 25,
  "translations": [ { "id": "...", "type": "voice", "name": "...", ... }, ... ]
}
```

### GET /translations
```
Параметры:
  - id: string
  - id_type: string

Ответ:
{
  "translations": [
    {
      "id": "1234",
      "type": "voice",
      "name": "Озвучка Рус1",
      "series_range": [1, 25]
    }
  ]
}
```

### GET /play
```
Параметры:
  - id: string
  - id_type: string
  - seria_num: integer (номер серии)
  - translation_id: string (ID озвучки)
  - quality: integer (720, 1080 и т.д.)

Ответ:
{
  "m3u8_url": "https://..."
}
```

## 🐛 Известные проблемы и решения

### Проблема: Видео не загружается
**Решение:**
1. Проверьте, что backend запущен
2. Откройте DevTools (F12) → Network
3. Проверьте ошибки в Console
4. Убедитесь, что озвучка доступна

### Проблема: Озвучки не отображаются
**Решение:**
1. Не все контент может иметь несколько озвучек
2. Попробуйте другое аниме
3. Проверьте response от /episodes endpoint

### Проблема: Серии не переключаются
**Решение:**
1. Проверьте, что series_count > 0
2. Убедитесь, что выбрана озвучка
3. Проверьте ошибки в console

## 🎓 Как это работает

1. **Загрузка контента**
   - Пользователь выбирает аниме из каталога
   - App получает информацию и подготавливает параметры

2. **Открытие плеера**
   - При клике на Play срабатывает `playVideo()`
   - `setActiveVideo()` и `setUseFullscreenPlayer(true)`
   - VideoPlayer монтируется в DOM

3. **Загрузка озвучек и серий**
   - VideoPlayer загружает озвучки через `fetchTranslationOptions()`
   - VideoPlayer загружает серии через `fetchEpisodes()`
   - Отображает меню в интерфейсе

4. **Воспроизведение видео**
   - При выборе серии и озвучки вызывается `fetchPlayUrl()`
   - Получается m3u8 ссылка от Kodik парсера
   - HLS.js загружает и воспроизводит видео

5. **Управление**
   - Клавиатура, мышь и клавиши TV-пульта работают одинаково
   - Все события обрабатываются в компоненте
   - Состояние сохраняется локально и в Firestore
